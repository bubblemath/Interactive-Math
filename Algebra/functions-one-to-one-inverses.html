Is it a function — and can you run it backward?

Two questions decide everything here. Is it a function? — one output per input, which the vertical line test checks on a graph and the input column checks in a table. Is it one-to-one? — one input per output, checked by the horizontal line test, the output column, and ultimately confirmed when two functions compose back to x.

01

Two questions, four tools

Everything on this page is one of these two checks in disguise.

Is it a function?

Every input is allowed exactly one output. Nothing in the domain maps to two different values.

Graph: the vertical line test — no vertical line may cross more than once.
Table: no input value repeats with a different output.

Is it one-to-one?

Different inputs always give different outputs — so the rule can be reversed without ambiguity.

Graph: the horizontal line test — no horizontal line may cross more than once.
Table: no output value repeats.
Inverse: if one-to-one, an inverse exists — confirm it with composition.

A one-to-one function has an inverse that undoes it. The test is composition in both directions: f(g(x)) = x and g(f(x)) = x. If only one direction works, they are not inverses. And a function that fails the horizontal line test — like x² — can still earn an inverse if you restrict its domain so it becomes one-to-one.
