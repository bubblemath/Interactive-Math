🧊 Concrete Model (Base-10 Blocks)

The Concept

Students use physical objects to multiply. This connects multiplication to the repeated addition model using manipulatives.

From your materials: "Students use physical objects to add" — the same principle applies to multiplication!

"How would you help a student transition from counting individual blocks to understanding the efficiency of grouping by tens and hundreds?"

📚 Teaching Tip

  • Start with single-digit × single-digit using unit cubes
  • Introduce longs (tens) when products exceed 9
  • Show regrouping: 10 units → 1 long, 10 longs → 1 flat

📝 Expanded Algorithm

The Concept

The expanded algorithm explicitly shows the distributive property at work. It breaks numbers into place values and multiplies each part.

From your materials: 4 × 12 using the Distributive Property: 4 × (10 + 2) = (4 × 10) + (4 × 2) = 40 + 8 = 48

🤔 Teaching Connection

"The expanded algorithm makes the distributive property visible. How does this help students understand WHY the standard algorithm works?"

📚 Teaching Tip

  • Emphasize place value: 12 = 10 + 2, not just "1 and 2"
  • Write out the distributive property explicitly
  • This bridges concrete models to abstract algorithms
  • Great for students who need to see "why" before "how"

The Concept

The partial products algorithm breaks multiplication into smaller, manageable pieces. Each partial product is calculated separately, then added together.

📐 Area Model (Box Method)

The Concept

The area model represents multiplication as finding the area of a rectangle. The rectangle is decomposed into smaller rectangles based on place value.

This connects multiplication to geometry and provides a visual proof of the distributive property.

  • This method extends naturally to algebra (FOIL)

🔲 Lattice Method

The Concept

The lattice method organizes multiplication in a grid where diagonals automatically handle place value. Each cell contains a single-digit multiplication, split by a diagonal into tens (top-left) and ones (bottom-right).

  • Setup: First number across the TOP, second number down the RIGHT side
  • Diagonals: Lines go from top-right to bottom-left in each cell
  • Reading: Add along diagonals, read from top-left down and across bottom

Step-by-Step Process
  1. Draw the grid: Columns = digits in first number, Rows = digits in second number
  2. Place numbers: First number across TOP, second number down RIGHT side
  3. Draw diagonals: Each cell gets a line from top-right corner to bottom-left corner
  4. Multiply: Top digit × Right digit for each cell. Tens go top-left, ones go bottom-right
  5. Add diagonals: Start from bottom-right, add along each diagonal
  6. Carry if needed: If diagonal sum ≥ 10, carry to next diagonal
  7. Read answer: From top-left down the left side, then across the bottom

🤔 Teaching Connection

"Why might the lattice method help students who struggle with the standard algorithm? What does the diagonal represent mathematically?"

📚 Teaching Tip

  • Each cell only requires single-digit multiplication (times table up to 9)
  • Diagonals group digits by place value automatically
  • Great for students who struggle with keeping track of carries
  • Works well for larger numbers (3+ digits)

✍️ Standard Algorithm

The Concept

The standard algorithm is the traditional, compact method for multiplication. It's efficient but requires understanding of place value and carrying.

  • Setup: Larger number on top, aligned to the right
  • Carries: Written above the top number (cross out after each row)
  • Placeholders: Add zeros when multiplying by tens, hundreds, etc.

🎓 Educator's Lab

Teaching Scenarios

Practice matching algorithms to student needs. Select the best approach for each situation.

Scenario 1: The Concrete Thinker

A student can multiply single digits but gets confused with 4 × 12. They say "I don't understand where the 48 comes from."

What would you try first?

Show 4 groups of 12 using base-10 blocks
Teach the standard algorithm with carrying
Have them memorize 4 × 12 = 48

Scenario 2: The Carrying Struggler

A student understands multiplication conceptually but makes errors when carrying in the standard algorithm. They often forget to add the carried digit or put it in the wrong place.

What algorithm might help?

More practice with the standard algorithm
Lattice method (organizes carries automatically)
Go back to single-digit multiplication

Scenario 3: The "Why" Asker

A student asks "Why do we write a zero when multiplying by the tens digit?" They want to understand the reasoning, not just follow steps.

What would help most?

Tell them "That's just how it works"
Show the expanded algorithm with distributive property
Have them practice more problems

Scenario 4: The Visual Learner

A student learns best through pictures and diagrams. They're struggling to see how 23 × 45 breaks apart.

What representation would help?

Area model showing the rectangle decomposition
Verbal explanation of the steps
Written list of partial products
